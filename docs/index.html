Use it in a regular web page

Two lines from the jsDelivr CDN:

<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/jozefbenko/solargis-dsc-mini@main/docs/tokens.css">
<script type="module" src="https://cdn.jsdelivr.net/gh/jozefbenko/solargis-dsc-mini@main/docs/wc.js"></script>

Use it in a Claude artifact

Different story. Claude artifacts run in a sandboxed iframe whose CSP only allows scripts and styles from cdnjs.cloudflare.com and claudeusercontent.com (plus fonts.googleapis.com for styles). jsDelivr and GitHub Pages are both blocked.

The working pattern is inline injection: the MCP injects the contents of tokens.css and wc.js into the artifact-rendering Claude's system prompt, which pastes them as inline <style> and <script> blocks. 'unsafe-inline' is permitted in the CSP, so this works reliably. See artifact-claude.md for the full preamble template.

Then use any <sg-*> tag below. See a full composition in the PV site yield estimator example.
